SAL-CP EXPERIMENT

Self-Aware LLM Communication Protocol - An experimental, rich context communication protocol for collaborative AI agents. Research into contextual agent-to-agent messaging.

‚ö†Ô∏è Experimental Research - This project explores rich context communication and will remain permanently in experiment status.

üîç Research Problem

Current agent communication is often stateless and context-poor, leading to misinterpretations and errors like the "lost in the middle" problem. An agent receiving a prompt lacks critical context: Why is this task being requested? What is the sender's current state?

üí° Research Approach: Self-Aware, Contextual Communication

SAL-CP defines a standardized message envelope that wraps every interaction, providing rich, structured context.

SAL-CP Message = Header (State & Capabilities) + Payload (The Task)

This allows an agent to understand not just what to do, but why, and to self-assess the quality of information it receives.

üìã Formal Specification

Research into clear JSON schema defining message structure with standardized fields for cognitive state, context, and performance metadata.

üß† Cognitive State Awareness

Experimental fields for agents to communicate internal state: current_load, attentional_focus, confidence levels, and processing capacity.

üéØ Contextual Pointers

Research mechanism to defeat "lost in the middle" problem by allowing agents to point to critical information within large contexts.

üìä Performance Metadata

Experimental system for agents to report confidence, ambiguities, and processing metrics, enabling trust-based collaboration frameworks.

üìö Reference Library

Research implementation providing simple tools to construct, parse, and validate SAL-CP messages across different agent systems.

üîó EAX Integration

Designed as underlying communication layer for EAX Marketplace, where Task_Broadcasts become specialized SAL-CP messages.

Research Example

Exploring how rich context can improve agent collaboration accuracy and reliability.

# Creating a rich, contextual request
from sal_cp import Message, Header, Payload

# The sender agent describes its own state
header = Header(
    sender_agent_id="Orchestrator-001",
    cognitive_state={
        "current_load_percent": 75,
        "attentional_focus": ["sales_data"]
    }
)

# The payload contains task AND context
payload = Payload(
    primary_instruction="Summarize the attached document.",
    smart_summary="The user is a stressed CEO looking for risks in a Q3 sales report."
)

message = Message(header=header, payload=payload)
# Rich context enables more intelligent responses
